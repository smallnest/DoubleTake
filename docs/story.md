
/prd 使用Go语言实现一个命令行网络游戏程序：谁是卧底。 
因为是要摸鱼用，所以伪装成命令行的程序，不要花里胡哨的界面设计，重点在于游戏逻辑和网络通信的实现。

以下是规则和具体步骤：

《谁是卧底》是一种比拼语言表述与逻辑推理的聚会游戏。玩家分为平民（多数）和卧底（少数，可能含白板），分别拿到相近词语，通过每轮描述词语并投票投出可疑人员，最终平民需投死所有卧底获胜，卧底则需存活到最后或伪装身份（白板猜测平民词语）。

具体规则如下：
裁判： 负责开房间，提供密码给玩家，主持游戏。
角色分配：一般为4-10人。多数人拿到词语A（平民），1-3人拿到相关词语B（卧底）。高级玩法中可加入“白板”，即拿到空白卡片，不知道词语。
每轮描述：每位玩家轮流用一句话描述自己拿到的词语，不能直接说出词语本身，且不能让卧底轻易发现，同时需给同伴暗示。
投票环节：所有人描述完毕后进行投票，公认最可疑的（可能是卧底的）人出局。
平票与特殊情况：得票数最多的人出局；如果最高票数平票，通常保留两人，进入下一轮描述。
游戏结束与胜利条件：
平民胜利：如果在所有卧底出局前，卧底人数少于等于平民人数，则平民获胜。
卧底胜利：若最后仅剩3人（且包含卧底）时，卧底获胜。
白板胜利：若白板在游戏结束前猜出平民的词语，则白板单项获胜。 

提示：游戏切忌词语描述太明显，最好能给队友暗示但又混淆视听。


具体步骤：
1. 某人以裁判的身份启动程序
  - 设定玩家的人数（4-10 人）、卧底的人数（1-3 人）、白板的人数(可以是 0)
  - 生成一个房间密码（根据自己的IP和端口+随机数 hash 生成），等待玩家加入
  - 等所有的玩家加入后，输入两个词语(平民的词语和卧底的词语)，随机分配角色（平民、卧底、白板）并发送相应的词语给每个玩家
2. 开始一轮，每个玩家发送描述自己的词语的文字
3. 裁判说投票，每个玩家输入要投票的玩家的名字
4. 统计投票结果，宣布被投票出局的玩家，并更新游戏状态
5. 重复步骤2-4，直到游戏结束（平民胜利、卧底胜利或白板胜利）

玩家加入的流程：
1. 玩家启动程序，输入房间密码加入游戏
2. 玩家输入自己的名字，等待裁判开始游戏
3. 游戏开始后，玩家根据裁判发送的词语进行描述，并参与投票环节
